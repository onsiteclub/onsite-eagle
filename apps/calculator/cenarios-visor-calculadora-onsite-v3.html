Cenários de Visor — Calculadora OnSite

Diretriz de implementação. Oito volumes cobrindo os tipos de cálculo que a calculadora suporta, com um mockup do visor para cada sub-cenário.

Importante para quem vai implementar: os mockups deste documento são de estrutura, não de estilo. Mostram apenas o que aparece no visor e onde (hierarquia, posição, tamanho relativo). Fundo branco, texto preto e bordas cinzas são escolhas neutras do wireframe — não da calculadora. O estilo visual real (fundo creme #f5f2ea, texto #1a1612, accent #EF9F27, fonte JetBrains Mono, visor com borda suave, teclas brancas com sombra) está definido no arquivo mockups-calculadora-onsite-v2.html. Combine os dois: este documento dá a estrutura, o outro dá o estilo.

Princípios universais do visor

Toda composição do visor respeita quatro regras, em qualquer volume:

  1. Uma única região retangular no topo da tela. Sem cards empilhados, sem abas, sem modos.
  2. Alinhamento à direita de toda a informação numérica (armação, resultado, labels). Tradição de calculadora.
  3. Hierarquia por tamanho: armação pequena em cima, resultado grande embaixo. Quando há dois resultados, ambos têm peso visual igual.
  4. Labels de unidade são minúsculos e ficam acima do valor que rotulam. Só aparecem quando necessários para desambiguar (caso com múltiplas unidades).

Volume 1 — Escalar puro

Números sem dimensão, sem unidade de construção. Base da aritmética geral. Cobre soma, subtração, multiplicação, divisão, precedência e parênteses.

4

Caso base. Esqueleto do qual todos os outros variam.

1.2Números grandes com separador de milhar

|valor| ≥ 1000 recebe separador do locale. Detectado pelo idioma do device (PT-BR: ponto; EN-CA: vírgula). Normalização acontece tanto na armação quanto no resultado.

Input aceita * e / (teclado). Armação exibe × e ÷ (símbolos matemáticos). Tradução é apenas visual; internamente o parser continua com ASCII.

1.5Parênteses explícitos

Armação preserva parênteses tal como o usuário digitou. Nenhuma reescrita. Comunica que a calculadora entende agrupamento (ganho sobre a versão anterior, que nem tinha parênteses).

Input
(2 + 3) * 4
Armação
(2 + 3) × 4
Resultado
20
4 + 4 × 10
44

1.6Precedência implícita invisível

PEMDAS é aplicado mas não exposto. Armação não injeta parênteses. Usuário que conhece a regra espera o comportamento; usuário que não conhece pode estranhar o resultado — para esse caso existe a tela secundária de "como foi armado", fora do escopo deste documento.

Input
4 + 4 * 10
Armação
4 + 4 × 10
Resultado

Volume 2 — Frações puras

Frações sem unidade de construção. Motor sempre preserva racional exato (fraction.js). Display escolhe formato por regra determinística: fração simplificada, número misto quando ≥ 1, inteiro quando o resultado for inteiro.

1/2 + 1/4
3/4

2.1Soma de frações próprias

Ambas as frações são < 1. Resultado < 1 exibido como fração simplificada. Sem decimal, sem número misto (não faz sentido abaixo de 1).

Input
1/2 + 1/4
Armação
1/2 + 1/4
Resultado
3/4
Regra
|v| < 1 → fração simplificada
5 1/2 + 3 1/4
8 3/4

2.2Soma de números mistos

Resultado ≥ 1 com parte fracionária → formato de número misto. Parte inteira e fração separadas por espaço. Nunca fração imprópria (35/4).

Input
5 1/2 + 3 1/4
Armação
5 1/2 + 3 1/4
Resultado
8 3/4
Regra
|v| ≥ 1 com parte fracionária → número misto
3/2 + 3/2
3

2.3Fração que resulta em inteiro

Resultado inteiro → exibe só o número, sem 3/1 ou 3,0. Cai no mesmo formato do Volume 1.1.

Input
3/2 + 3/2
Armação
3/2 + 3/2

2.4Fração irredutível com denominador grande

Resultado exato 8/15 é preservado. Em alguns casos denominador pode ser muito grande (ex: 1/7 + 1/11 = 18/77). Mostrar mesmo assim — é a representação exata. Alternativa decimal fica no Volume 1.4 (dízima com ≈).

Volume 3 — Comprimento imperial

Medidas em pés e polegadas, com ou sem frações. Cobre o padrão de construção norte-americano. Resultado sempre em formato X' Y Z/W" quando magnitude justifica; caso contrário só em polegadas com fração.

2' 6" + 3' 6"
6' 0"

3.1Soma feet-inches canônica

Resultado em feet-inches. Zero polegadas é explícito (6' 0", não 6') para consistência visual — sempre duas partes.

Input
2' 6" + 3' 6"
Armação
2' 6" + 3' 6"
Resultado
6' 0"
Regra
|v| ≥ 12" → feet-inches; sempre mostra ambas as partes
2'-6" + 1'-6"
4' 0"

3.2Notação dash-separated no input

Convenção de planta técnica (2'-6" = 2 pés 6 polegadas, não subtração). Motor v2 já resolve isso. Armação normaliza o dash para espaço.

Input
2'-6" + 1'-6"
Armação
2' 6" + 1' 6" (dash normalizado)
Resultado
4' 0"
Regra
pré-tokenizer substitui N'-M" por N' M"
5 1/2" + 3 1/4"
8 3/4"

3.3Polegadas com fração, resultado < 12"

Magnitude pequena — não converter para feet-inches. Só polegadas com número misto. Fração arredondada para granularidade de construção (1/16").

Input
5 1/2" + 3 1/4"
Armação
5 1/2" + 3 1/4"
Resultado
8 3/4"
Regra
|v| < 12" → só polegadas; |v| ≥ 12" → feet-inches
7' 2 1/2" + 0' 2 1/8"
7' 4 5/8"

3.4Feet-inches com fração preservada

Combinação mais rica: feet + polegadas + fração. O resultado mantém as três partes. Fração arredondada para 1/16" mais próximo; se o arredondamento alterar o valor, mostrar ≈ na dica.

Input
7' 2 1/2" + 0' 2 1/8"
Armação
7' 2 1/2" + 0' 2 1/8"
Resultado
7' 4 5/8"
Regra
arredondar fração residual para 1/16" mais próximo
2' − 3'
−1' 0"

3.5Resultado negativo

Subtração que vira negativa. Sinal precede o valor inteiro, não é repetido nas partes. Usa minus sign tipográfico (−, U+2212), não hífen.

Input
2' - 3'
Armação
2' − 3'
Resultado
−1' 0"
Regra
sinal negativo aplicado ao conjunto, minus sign Unicode

Medidas em metros, centímetros, milímetros. Motor escolhe unidade de display por magnitude — nunca mistura. Vírgula decimal respeita locale do device.

Caso canônico para usuário brasileiro/português. Vírgula como separador decimal preservada (motor v2 já normaliza internamente). Resultado inteiro mostrado sem casas decimais.

Matriz de decisão — motor → visor

Regra única que determina qual layout o visor renderiza. Motor retorna objeto CalculationResult com flags; UI aplica a matriz.

dim mixedSystems isError Layout Volumes
0—false Resultado único 1, 2
1falsefalse Resultado único (imperial OU métrico) 3, 4
1truefalse Resultado duplo, labels = sistemas 7.1, 7.2
≥ 2qualquerfalse Resultado duplo, labels = unidades 5, 6, 7.3
——true Estado de erro (palavra + mensagem) 8

Contrato do motor (para a UI consumir)

Objeto CalculationResult que o motor retorna. UI é função pura result → visor.

type CalculationResult = {
  // sempre presente
  isError: boolean;
  expression: string;          // armação normalizada (× ÷ − unicode)

  // quando isError = false
  dim: 0 | 1 | 2 | 3;           // escalar, comprimento, área, volume
  mixedSystems: boolean;        // true se input misturou imperial + métrico

  primary: {
    value: string;              // "5' 0"", "96", "8 3/4", "6 m"
    unitLabel: string | null;   // "sq ft", "imperial", null quando óbvio
  };

  secondary: {                  // só quando dim ≥ 2 OU mixedSystems
    value: string;
    unitLabel: string;
  } | null;

  isApproximate: boolean;       // true quando arredondamento alterou o valor
  exactForm: string | null;     // "10/3" para dízimas; null se desnecessário

  // quando isError = true
  errorMessage: string | null;  // "expressão incompleta", "divisão por zero"
  errorPosition: number | null; // índice de caractere para UI destacar
}
